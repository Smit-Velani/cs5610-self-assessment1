1. Top 10 Most Useful HTML Tags

These are the HTML tags I find most useful when building web pages:

  • <div> Container for grouping elements
  • <p> Paragraph text
  • <a> Hyperlinks to other pages
  • <img> Display images
  • <ul> / <li> Unordered lists
  • <form> User input forms
  • <input> Text fields, checkboxes
  • <nav> Navigation bar
  • <section> Page sections
  • <header> Page or section header

Sub-section 1.1: Semantic HTML

Semantic tags like <header>, <nav>, <main>, <section>, and <footer> help browsers and screen readers understand page structure.

Sub-section 1.2: Forms and Input

Forms are essential for collecting user data. The <form>, <input>, <label>, and <button> tags work together.

Sub-section 1.3: Media Tags

Tags like <img>, <video>, and <audio> allow rich media content on pages.

Sub-section 1.4: Linking and Navigation

The <a> tag is fundamental for all navigation. Combined with <nav> it creates clear site structure.

2. Important SEO Header Tags

After researching SEO best practices, here are the most important meta and header tags:

  • <title> Page title shown in browser tab and Google results — most important SEO tag
  • <meta name="description"> Short description shown under title in search results
  • <meta name="keywords"> Keywords related to your page content
  • <meta name="robots"> Controls whether search engines index the page
  • <link rel="canonical"> Tells Google which URL is the main version to avoid duplicates
  • <meta property="og:title"> Open Graph title for social media sharing
  • <meta property="og:description"> Description when shared on social media
  • <h1> Main heading — should contain the primary keyword

5. Font Sizes — Inline Styling

Same text with 6 different font sizes using both absolute and relative units:

  • This text is 10px — Absolute unit. Use when you need a fixed size that never changes regardless of screen or user settings. Example: small print disclaimers.
  • This text is 16px — Absolute unit. Browser default body text size. Good for fixed UI elements like buttons.
  • This text is 1rem — Relative to root font size (usually 16px). Use for body text so it scales with user browser preferences for accessibility.
  • This text is 1.5rem — Relative unit. Use for subheadings. Scales proportionally with the root size.
  • This text is 2em — Relative to parent font size. Use inside components where size should scale relative to the parent container.
  • This text is 3rem — Relative unit. Use for large display headings or hero text that should scale with the page.

6. CSS Positioning Options

Demonstrating 4 different positioning options: static, relative, absolute, and fixed (footer).

Static (default)
Normal document flow. No top/left/right/bottom. Use when you don't need special positioning.
Relative
Moved 20px down, 30px right from its normal position. Use when you need slight adjustments without affecting other elements.
Absolute
Positioned at top-right of its container. Use for tooltips, dropdowns, badges — elements that float over content.
Sticky
Sticks to top when scrolling past it. Use for section headers or sidebars that should stay visible while scrolling.

Note: The footer uses position: fixed — it stays at the bottom of the screen at all times regardless of scroll position. This is the most common use case for fixed positioning.
